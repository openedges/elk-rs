# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in elk-rs, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please email: **luuvish@gmail.com**

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact

We will acknowledge your report within 5 business days and work with you on a fix before any public disclosure.

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.11.x  | Yes       |
| < 0.11  | No        |

## Scope

This policy applies to the elk-rs Rust library, WASM module, and NAPI bindings. Vulnerabilities in third-party dependencies should be reported to their respective maintainers.
